찬희는 모든 상호작용에서 **도식화 최신화**를 극도로 선호한다.

필수 도식 유형 4가지:
1. **계층도** (Hierarchy) — 시스템 구조, 모듈 관계, 의존성
2. **논리흐름** (Logic Flow) — 데이터 파이프라인, 함수 호출 체인, 조건 분기
3. **시계열** (Timeline) — 작업 진행, Phase 체크포인트, 버전 변화
4. **워크플로우** (Workflow) — 실행 단계, 입출력, 검증 게이트

**Why:** 찬희의 사고방식이 시각적. 텍스트만으로는 전체 그림이 안 그려짐. 도식이 있어야 즉시 파악 + 의사결정 가능.
**How to apply:**
- 모든 세션에서 현황 보고 시 위 4가지 도식 중 2개 이상 포함
- 방대하고 상세하게 — "더 해줘"가 기본값
- ASCII 테이블, 플로우차트, 트리 구조 적극 사용
- 작업 완료 시 before/after 비교 도식 필수
