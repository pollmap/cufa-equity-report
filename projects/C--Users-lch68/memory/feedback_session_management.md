마라톤 세션에서 작업 누락과 콘텐츠 유실 방지 규칙.

**Why:** Claude Code Insights 분석(2026.03.27) 결과, 18세션 중 wrong_approach 18건, buggy_code 13건, missed_features 다수 발생. 원인은 세션당 10-18개 작업 동시 요청으로 인한 scope overload. 또한 리팩토링 중 ~2,141자 유실 사고 발생.

**How to apply:**
- 세션당 3~5개 관련 작업으로 제한. "이 세션에서는 이 3가지만 한다" 명시
- 멀티 작업 시 반드시 번호 체크리스트 유지, 완료 시마다 ✅ 업데이트
- 리팩토링/대규모 편집 시 전후 글자수/줄수 비교 필수 (삭제된 내용 없는지 확인)
- 대형 세션은 GSD 워크플로우 스킬 사용 권장
