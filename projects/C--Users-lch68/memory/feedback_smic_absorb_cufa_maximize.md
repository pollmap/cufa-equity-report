CUFA 보고서 전략: "SMIC의 글 + CUFA의 기술 = S+"

**SMIC에서 흡수 (약점 보완):**
- 텍스트 밀도: 12.5px/1.6 (기존 14px/1.8에서 압축)
- 문체 변주: "동사" 70% + 기업명 15% + 주어생략 15% (100% 동사 금지)
- bold 비율: 80% (100% 기계적 패턴 금지)
- 데이터 밀도: 문단당 수치 3-6개 (기존 2-4개)
- 차트-본문 밀착: 논거 직후 차트 배치 (파트 끝 몰아넣기 금지)
- 추론 5단계: 트렌드→구조변화→포지셔닝→수익영향→리레이팅
- 비유 절제: 핵심 3개만 (과잉 장식 금지)

**CUFA 극대화 (고유 강점):**
- 인터랙티브 HTML (progress bar, sticky header, float TOC, expand card)
- 다크 테마 + 시그니처 퍼플 #7c6af7
- 4중 밸류에이션 (PSR+DCF+Reverse DCF+Football Field)
- Kill Conditions (투자논지 사망 조건)
- Investment Thesis Dashboard (6-Box Grid)
- counter_arg 시각 분리 (빨간 우려 / 초록 반박)
- 반응형 3단계 + 인쇄용 CSS

**Why:** 2026.04.04 이노스페이스 보고서 빌드 과정에서 SMIC 5편+YIG 7편과 비교 리뷰 결과 도출. CUFA 종합 7.0 vs SMIC 7.6 → "사람이 쓴 느낌"이 최대 갭.

**How to apply:** 모든 CUFA 보고서 빌드 시 이 기준 적용. build_template.py CSS 기본값으로 반영 완료. 문체 규칙은 서브에이전트 프롬프트에 필수 포함.
