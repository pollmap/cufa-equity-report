## 근본적 순서 오류 (가장 중요한 교훈)

**문제**: HTML 보고서를 먼저 만들고 엑셀을 나중에 만들었다. 데이터가 추정치+하드코딩 범벅.

**올바른 순서 (v11 확정)**:
1. MCP 데이터 수집 (DART IS/BS/CF, 주가, 매크로)
2. 추가 리서치 (사업보고서 정독, 거래처, 항로, 뉴스)
3. **엑셀 시트 빌드 먼저** (수집한 실제 데이터 기반)
4. 본문 작성 (엑셀 참조하며)
5. HTML 빌드
6. Evaluator 검증

**Why:** 엑셀이 Single Source of Truth여야 한다. 보고서의 모든 숫자는 엑셀에서 온다.

## 추가 교훈

- **Appendix = 표만** (시각화 금지)
- **단락 길이 균일화**: SMIC은 200~400자/단락, 페이지당 800~1200자. 특정 단락이 너무 길거나 짧으면 안 됨
- **"CUFA 평가/추정" 표현 금지** 강화 필요
- **화이트 테마 확정** (v11~)
- **사이드바 키워드 시각적 구분** 필요 (border-right가 안 보임)
- **CSI 시계열 차트** 반드시 포함 (현재 수치만으로 부족)

**How to apply:** 다음 보고서부터 엑셀 먼저 → HTML 나중 순서로.
