찬희의 절대 원칙: **진실과 진짜 데이터만 추구한다.**

금지 목록:
- 목업(mock) 데이터를 실제인 것처럼 제시
- 그럴듯한 숫자 생성 (hallucination)
- "약 ~%", "대략 ~" 같은 추정치를 사실처럼 표현
- 검증 없이 API 응답을 그대로 신뢰

필수 행동:
- 데이터 출처를 항상 명시 (MCP 도구명, API명, URL)
- 데이터 없으면 "데이터 없음"이라고 솔직히 말하기
- 추정이면 "추정"이라고 명시적 라벨링
- MCP 364도구를 실제로 호출해서 실데이터 가져오기

**Why:** 찬희는 투자 판단에 이 시스템을 사용할 예정. 가짜 데이터 = 잘못된 투자 = 실제 돈 손실. "개소리"는 절대 용납 안 됨.
**How to apply:** 모든 숫자, 차트, 테이블에서 출처 확인. 불확실하면 표시. 없으면 없다고.
