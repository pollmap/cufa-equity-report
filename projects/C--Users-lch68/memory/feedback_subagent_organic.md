서브에이전트를 극한으로 활용해라. 병렬로 돌릴 수 있는 건 무조건 병렬로. 그래야 일찍 끝난다.

**Why:** 순차 작업은 시간 낭비. 5개 섹션 파일을 하나씩 수정하면 5배 느림. 병렬이면 1배.

**How to apply:**
- 독립적인 파일 수정은 무조건 병렬 에이전트 (섹션 5개 → 에이전트 5개)
- 읽기 작업도 병렬 (3개 파일 읽기 → Explore 에이전트 3개)
- 단, 의존성 있는 작업은 순차 (config.py 먼저 → sections/*.py 나중에)

---

유기적 연결이 핵심. 단순히 텍스트 적는 걸로 끝내면 안 됨. SMIC이 89점인 이유가 바로 이 유기적 흐름.

**Why:** SMIC S-Oil은 "전술한 바와 같이~", "[투자포인트 1]에서 전술한 것처럼~" 등으로 섹션 간을 유기적으로 연결. CUFA는 이게 없어서 섹션이 독립적으로 읽히고 흐름이 끊김.

**How to apply:**
- 본문 작성 완료 후 **2차 검증 에이전트** 돌리기
- 검증 항목: ① '전술한/후술할' 연결사 2회+ 있는지 ② 산업분석→투자포인트→밸류에이션 논리 체인이 끊기지 않는지 ③ 같은 수치가 여러 섹션에서 일관적인지
- 빌드 후 HTML을 읽어서 "섹션 A의 결론이 섹션 B의 전제로 이어지는가" 검증
