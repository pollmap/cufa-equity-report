# /deep-review — 코드 종합 검토 (12-Point Checklist)

지정된 파일/디렉토리에 대해 12개 항목 종합 코드 리뷰 수행.

## 사용법
```
/deep-review [파일경로 또는 디렉토리]
```

## 12-Point 체크리스트

### 1. 목적 부합성
- 구현이 원래 의도대로 동작하는가?
- 누락된 기능이나 엣지 케이스는?

### 2. 잠재적 버그
- null/None 체크 누락
- off-by-one 에러
- 타입 불일치
- 예외 처리 누락/과잉

### 3. 보안 문제
- 입력 검증 (injection, XSS, path traversal)
- 인증/권한 체크
- 민감 정보 노출 (str(e) 반환, 로그에 비밀 기록)
- subprocess 호출 시 shell=True 사용 여부

### 4. 큰 함수/파일 분할
- 함수 100줄 초과 → 분할 검토
- 파일 500줄 초과 → 모듈 분리 검토
- 중첩 깊이 4단계 초과 → 리팩토링

### 5. 코드 재사용
- 중복 코드 3곳 이상 → 유틸리티 추출
- 기존 BaseMCPServer, CacheManager, RateLimiter 활용 여부
- 기존 어댑터 패턴과 일관성

### 6. 사이드이펙트
- 글로벌 상태 변경
- 파일 시스템 쓰기
- 네트워크 호출
- 스레드 생성
- 각 사이드이펙트가 문서화/예측 가능한지

### 7. 전체 변경사항 일관성
- 네이밍 컨벤션 통일
- 에러 응답 형식 통일 ({"error": True, "message": ...})
- 로깅 레벨 적절성

### 8. 불필요한 코드 정리
- 사용되지 않는 import
- 주석 처리된 코드
- 도달 불가능한 코드
- 불필요한 변수

### 9. 코드 품질
- 타입 힌트 일관성
- 독스트링 존재 (public 함수)
- 매직 넘버 → 상수 추출

### 10. 사용자 흐름
- API 호출 순서가 자연스러운가
- 에러 메시지가 사용자에게 유용한가 (한국어)
- 타임아웃/재시도 정책

### 11. 연관 부분 체크
- 변경된 함수를 호출하는 곳 모두 확인
- import 경로 변경 시 모든 참조 확인
- 설정 파일/환경 변수 의존성

### 12. 배포 준비 확인
- systemd 서비스 재시작 필요 여부
- 환경 변수 추가/변경 여부
- 마이그레이션 필요 여부
- 롤백 가능한 변경인지

## 출력 형식

각 항목에 대해:
```
| # | 항목 | 결과 | 발견 사항 |
|---|------|------|----------|
| 1 | 목적 부합성 | PASS/WARN/FAIL | 상세 |
...
```

FAIL 항목은 즉시 수정 코드와 함께 제시.
WARN 항목은 개선 방안 제안.
